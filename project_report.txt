# Istanbul Guided Recommendation System - Project Report

## 1. Project Overview
- **Name:** Istanbul Guided Recommendation System
- **Purpose:** AI-driven travel recommendation engine for Istanbul (places/events) using user interests + historic data.
- **Key outputs:** recommendations list (name, category, score, explanation), itinerary generation, rating + feedback integration.

---

## 2. Architecture

1. **Backends**
   - `api/app.py`: FastAPI REST API (port 8000)
   - `web/app.py`: Flask web frontend (port 5000)
2. **Core logic**
   - `recommendation/recommender.py`
   - `llm/explanation_generator.py`
3. **Data**
   - `data/places.json`, `data/events.json`
4. **DB**
   - `database/db_connection.py` (SQLite)
   - rating + visited + LLM cache tables
5. **Frontend**
   - `web/templates/base.html`, `recommendations.html`, `itinerary.html`
   - scoped UI, no "profile" menu (removed per request)

---

## 3. Recommendation Engine

- TF-IDF vectorization (tags + description)
- Cosine similarity vs user interest vector
- Hybrid score:
  - 70% similarity
  - 30% rating
- Support:
  - exclude visited
  - search by category, min rating
  - dynamic shuffle for variety

---

## 4. Explanation System

- `llm/explanation_generator.py`
- Primary: OpenAI GPT calls (prompt with category-specific rules)
- Fallback
  - local rules (`generate_ai_explanation()`)
  - deterministic for stable tests
- Caching: 1h TTL in DB to reduce calls
- New stricter prompt:
  - categories: food, mosque/religious, cultural, tour, nature
  - 2-3 sentences, 2 features, emotional tone, no "matches your interest"
